**Дата:** 09.04.2026  
**Домен:** azoneai.ru  
**Хостинг:** Yandex Cloud Object Storage + CDN  
**Стек:** Astro SSG, Tailwind CSS  
**Статус:** Сайт доступен из РФ ✅  

---

## 1. Executive Summary

Сайт azoneai.ru успешно переехал на Yandex Cloud — **критическая проблема с доступностью из РФ решена**. Это главное достижение: теперь можно двигаться к SEO и лидогенерации.

Фактический аудит dist-папки показывает **сильную техническую базу**: 22 страницы в sitemap, canonical-теги на всех страницах, Schema.org JSON-LD (Organization) на каждой странице, OG-теги, robots.txt с ссылкой на sitemap. Блог — 10 экспертных статей, 5 продуктовых страниц. Это серьёзный контентный актив.

**Три ключевых проблемы, которые нужно решить прямо сейчас:**

1. **Нет аналитики.** Ни Яндекс.Метрики, ни GA4 — а значит, нет данных о трафике, поведении и конверсиях.
2. **Слабые title и description на продуктовых страницах.** Пример: `<title>AzoneDoc | AZONE-AI</title>`, description: «Интеллектуальное управление документами» — это 5 слов, поисковик не поймёт, о чём страница.
3. **Изображения не попали в dist.** Папка `/images/` отсутствует в архиве — все `<img src="/images/...">` отдают 404. Это ломает визуал сайта и Core Web Vitals.

## 3. Что сейчас мешает SEO и маркетингу

### 3.1 КРИТИЧНО: Папка /images/ не в dist

Все страницы ссылаются на ~45 изображений в `/images/` — но в загруженном dist-архиве этой папки нет. Это значит:
- Логотип (`/images/logo.png`) — 404
- Серверная фотография (`/images/servers.webp`) — 404
- Все иллюстрации в статьях блога (16 webp-файлов) — 404
- Все иллюстрации продуктовых страниц (30+ webp-файлов) — 404

**Действие:** Убедиться, что `/images/` загружена в бакет Object Storage. Если она не включается в `astro build` автоматически — скопировать из `public/images/` в бакет вручную или добавить в deploy-скрипт.

**Ответственный:** Developer  
**Эффект:** Без этого сайт выглядит сломанным. Критично для первого впечатления и Core Web Vitals (LCP).

### 3.2 КРИТИЧНО: Нет аналитики

В HTML-коде нет ни одного счётчика:
- Нет Яндекс.Метрики
- Нет Google Analytics 4
- Нет подтверждения в Яндекс.Вебмастере
- Нет Google Search Console

**Действие:** Добавить коды счётчиков в `<head>` layout-файла Astro, пересобрать и задеплоить.

**Ответственный:** Developer + Marketing  
**Эффект:** Без аналитики невозможно оценить эффективность чего-либо.

### 3.3 КРИТИЧНО: Не зарегистрирован в вебмастер-инструментах

Даже при наличии sitemap — Яндекс и Google не знают о сайте, пока не подать sitemap вручную.

**Действие:**
1. Зарегистрировать azoneai.ru в Яндекс.Вебмастере (https://webmaster.yandex.ru)
2. Зарегистрировать в Google Search Console (https://search.google.com/search-console)
3. Подать sitemap в обоих инструментах
4. Подтвердить владение доменом (мета-тег или DNS-запись)

**Ответственный:** SEO  
**Эффект:** Начало индексации. Без этого — 0 органического трафика.

### 3.6 ВЫСОКИЙ ПРИОРИТЕТ: Sitemap без lastmod

Текущий sitemap не содержит `<lastmod>` — поисковики не знают, когда страницы обновлялись. Для блога это особенно важно: свежий контент получает приоритет в выдаче.

**Действие:** Добавить `lastmod` в конфигурацию `@astrojs/sitemap` или вручную через frontmatter-дату.

### 3.7 ВЫСОКИЙ ПРИОРИТЕТ: Нет директивы Host в robots.txt

Для Яндекса рекомендуется добавить:
```
Host: https://azoneai.ru
```

### 3.8 СРЕДНИЙ ПРИОРИТЕТ: Ранние статьи (01–06) имеют только 1 блок Schema.org

Статьи 01–06 содержат только Organization schema. Статьи 08–10 уже содержат Article/FAQ schema. Нужно дотянуть 01–07 до того же уровня.

### 3.9 СРЕДНИЙ ПРИОРИТЕТ: PDF-презентация не в dist

Ссылка «Скачать презентацию» ведёт на `/AZONE_AI_presentation.pdf`, но файл отсутствует в dist. Либо он не попал при сборке, либо его нужно загрузить в бакет отдельно.

### 3.10 СРЕДНИЙ ПРИОРИТЕТ: Нет хлебных крошек (BreadcrumbList)

На продуктовых страницах и в статьях блога нет breadcrumbs. Это улучшает навигацию и даёт расширенные сниппеты в поисковой выдаче.
