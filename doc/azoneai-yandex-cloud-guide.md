# Размещение сайта AZONE-AI на Yandex Cloud
## Пошаговый гайд для новичков

**Что получится в итоге:**
- Сайт azoneai.ru работает на Yandex Cloud Object Storage
- HTTPS через бесплатный сертификат Let's Encrypt
- Деплой одной командой из терминала Cursor
- Стоимость: ~100–300 руб./мес. (для статического сайта с небольшим трафиком)

**Что понадобится:**
- Аккаунт Яндекса (обычный, через Яндекс ID)
- Доступ к DNS-записям домена azoneai.ru (у регистратора домена)
- Терминал (встроен в Cursor)
- 1–2 часа времени

## Часть 1. Создание аккаунта Yandex Cloud

### Шаг 1.1. Регистрация

1. Открой https://console.yandex.cloud/
2. Войди через Яндекс ID (если нет — создай на passport.yandex.ru)
3. Нажми «Попробовать бесплатно»
4. Заполни данные организации (можно ООО «Азон» или личные данные)
5. Привяжи карту — Yandex Cloud даёт **стартовый грант** (на момент 2026 — обычно 4000 ₽, хватит на несколько месяцев)

> **Важно:** Деньги не спишутся сразу. Грант покрывает расходы на первые месяцы. Object Storage для статического сайта стоит копейки.

### Шаг 1.2. Создание каталога (folder)

После входа в консоль ты окажешься в облаке (cloud) с каталогом по умолчанию `default`. Можно работать в нём, можно создать отдельный:

1. Левое меню → **Все сервисы** → **Resource Manager**
2. Нажми «Создать каталог»
3. Назови: `azoneai-prod`
4. Нажми «Создать»

## Часть 2. Создание бакета Object Storage

### Шаг 2.1. Открой Object Storage

1. В консоли управления: левое меню → **Object Storage**
2. Нажми **«Создать бакет»**

### Шаг 2.2. Настройки бакета

Заполни поля:

| Поле | Значение | Комментарий |
|------|----------|-------------|
| **Имя бакета** | `azoneai.ru` | Важно: имя = домен! Точно так, с точкой |
| **Макс. размер** | Без ограничений (или 1 ГБ) | Для статики хватит |
| **Чтение объектов** | **Публичный** | Чтобы пользователи могли видеть сайт |
| **Чтение списка объектов** | **Публичный** | Нужно для хостинга |
| **Класс хранилища** | Стандартный | Для часто запрашиваемых файлов |

3. Нажми **«Создать бакет»**

### Шаг 2.3. Включение хостинга

1. Открой созданный бакет `azoneai.ru`
2. Слева → **Настройки** → вкладка **Веб-сайт**
3. В разделе **Хостинг** заполни:
   - **Главная страница:** `index.html`
   - **Страница ошибки:** `404.html` (если есть в проекте, иначе тоже `index.html`)
4. Нажми **«Сохранить»**

Появится ссылка вида: `http://azoneai.ru.website.yandexcloud.net` — это временный адрес, по нему сайт будет доступен сразу после загрузки файлов.

## Часть 3. Создание сервисного аккаунта и ключей

Для деплоя из командной строки нужен сервисный аккаунт с ключами доступа (аналог AWS Access Key).

### Шаг 3.1. Создание сервисного аккаунта

1. Левое меню → **IAM** (Управление доступом) → **Сервисные аккаунты**
2. Нажми **«Создать сервисный аккаунт»**
3. Имя: `azoneai-deployer`
4. Роли — нажми «Добавить роль» и выбери:
   - `storage.editor` (редактирование объектов в бакете)
5. Нажми **«Создать»**

### Шаг 3.2. Создание статического ключа доступа

1. Открой созданный сервисный аккаунт `azoneai-deployer`
2. Сверху → **«Создать ключ доступа»** → **«Статический ключ доступа»**
3. Описание: `s3 deploy key`
4. Нажми **«Создать»**

**ЗАПИШИ ОБА ЗНАЧЕНИЯ:**
- **Идентификатор ключа (Key ID)** — выглядит как `YCAJExxxxxxxxxxxxxxxx`
- **Секретный ключ (Secret Key)** — длинная строка, **показывается один раз!**

> **Сохрани секретный ключ** в надёжном месте. Если потеряешь — придётся создавать новый.

## Часть 4. Настройка инструмента деплоя (AWS CLI)

AWS CLI работает с Yandex Object Storage, потому что API совместим с Amazon S3.

### Шаг 4.1. Установка AWS CLI

Открой терминал в Cursor:

**macOS:**
```bash
brew install awscli
```

**Windows (PowerShell от имени администратора):**
```powershell
msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2.msi
```

**Linux:**
```bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
```

Проверка:
```bash
aws --version
```

### Шаг 4.2. Настройка профиля для Yandex Cloud

```bash
aws configure --profile yc
```

Введи по очереди:
```
AWS Access Key ID: YCAJExxxxxxxxxxxxxxxx     ← твой Key ID из шага 3.2
AWS Secret Access Key: xxxxxxxxxxxxxxxx       ← твой Secret Key из шага 3.2
Default region name: ru-central1
Default output format: json
```

### Шаг 4.3. Проверка подключения

```bash
aws s3 ls --endpoint-url=https://storage.yandexcloud.net --profile yc
```

Должен появиться список бакетов, в том числе `azoneai.ru`.

## Часть 5. Первый деплой сайта

### Шаг 5.1. Сборка Astro-проекта

В терминале Cursor, в папке проекта:

```bash
npm run build
```

Astro создаст папку `dist/` с готовыми HTML/CSS/JS-файлами.

### Шаг 5.2. Загрузка в бакет

```bash
aws s3 sync dist/ s3://azoneai.ru \
  --endpoint-url=https://storage.yandexcloud.net \
  --profile yc \
  --delete
```

Что делает эта команда:
- `sync dist/` — синхронизирует локальную папку `dist/` с бакетом
- `s3://azoneai.ru` — имя бакета
- `--endpoint-url` — указывает, что это Yandex Cloud, а не Amazon
- `--profile yc` — использует ключи, которые мы настроили
- `--delete` — удаляет из бакета файлы, которых нет в `dist/` (чистый деплой)

### Шаг 5.3. Проверка

Открой в браузере:
```
http://azoneai.ru.website.yandexcloud.net
```

Если видишь свой сайт — всё работает! Переходим к привязке домена.

## Часть 6. Привязка домена azoneai.ru

Есть два варианта DNS. Рекомендую **Вариант A** — через Yandex Cloud DNS, это проще и надёжнее.

### Вариант A: DNS через Yandex Cloud (рекомендуемый)

#### Шаг 6A.1. Создание DNS-зоны

1. Консоль → **Cloud DNS** (левое меню → Сеть → Cloud DNS)
2. Нажми **«Создать зону»**
3. Заполни:
   - **Имя зоны:** `azoneai-zone`
   - **Зона:** `azoneai.ru.` (с точкой в конце!)
   - **Тип:** Публичная
4. Нажми **«Создать»**

#### Шаг 6A.2. Добавление ANAME-записи

1. Открой созданную зону `azoneai.ru`
2. Нажми **«Создать запись»**
3. Заполни:
   - **Имя:** оставь пустым (это для корневого домена `azoneai.ru`)
   - **Тип:** `ANAME`
   - **TTL:** `600`
   - **Значение:** `azoneai.ru.website.yandexcloud.net`
4. Нажми **«Создать»**

> **Что такое ANAME?** Это специальный тип записи Yandex Cloud DNS, который позволяет привязать корневой домен (без www) к другому домену. Обычный CNAME для корневых доменов не работает — это ограничение стандарта DNS.

#### Шаг 6A.3. Делегирование домена на Yandex Cloud DNS

Перейди в панель управления **регистратора домена** (где покупал azoneai.ru — REG.RU, Nic.ru, Timeweb, RU-CENTER и т.д.) и замени NS-серверы на:

```
ns1.yandexcloud.net
ns2.yandexcloud.net
```

Где это делается (примеры):
- **REG.RU:** Мои домены → azoneai.ru → DNS-серверы → Изменить
- **Nic.ru:** Домены → azoneai.ru → Делегирование → NS-серверы
- **Timeweb:** Домены → azoneai.ru → DNS → Внешние DNS

> **Важно:** После смены NS-серверов обновление может занять от 15 минут до 48 часов (обычно 1–2 часа). В это время сайт может быть недоступен по домену.

### Вариант B: DNS у текущего регистратора

Если не хочешь переносить DNS в Yandex Cloud, можно добавить CNAME-запись у регистратора. Но есть ограничение: CNAME **не работает для корневого домена** (azoneai.ru), только для поддоменов (www.azoneai.ru).

Поэтому при таком варианте:

1. Создай CNAME-запись:
   - **Имя:** `www`
   - **Тип:** `CNAME`
   - **Значение:** `azoneai.ru.website.yandexcloud.net`

2. Основной адрес сайта будет `www.azoneai.ru`, а не `azoneai.ru`. Это менее удобно, поэтому **Вариант A предпочтительнее**.

## Часть 7. Настройка HTTPS (SSL-сертификат)

### Шаг 7.1. Выпуск сертификата Let's Encrypt

1. Консоль → **Certificate Manager** (левое меню → Безопасность → Certificate Manager)
2. Нажми **«Добавить сертификат»** → **«Let's Encrypt сертификат»**
3. Заполни:
   - **Имя:** `azoneai-cert`
   - **Домены:** `azoneai.ru`
   - **Тип проверки:** `DNS` (рекомендуется, если DNS уже в Yandex Cloud)
4. Нажми **«Создать»**

### Шаг 7.2. Прохождение проверки домена

Если DNS в Yandex Cloud (Вариант A) — проверка пройдёт **автоматически** за несколько минут.

Если DNS у стороннего регистратора — нужно будет добавить TXT-запись, которую покажет Certificate Manager. Инструкция появится на странице сертификата.

Статус сертификата сменится на **«Issued»** (Выпущен). Это может занять 5–30 минут.

### Шаг 7.3. Привязка сертификата к бакету

1. Вернись в **Object Storage** → бакет `azoneai.ru`
2. **Настройки** → вкладка **HTTPS**
3. Источник: **Certificate Manager**
4. Выбери сертификат `azoneai-cert`
5. Нажми **«Сохранить»**

После этого:
- Сайт доступен по `https://azoneai.ru`
- HTTP-запросы автоматически перенаправляются на HTTPS

## Часть 8. Автоматизация деплоя

### Шаг 8.1. Добавление скрипта в package.json

Открой `package.json` в корне Astro-проекта и добавь скрипт:

```json
{
  "scripts": {
    "dev": "astro dev",
    "build": "astro build",
    "preview": "astro preview",
    "deploy": "astro build && aws s3 sync dist/ s3://azoneai.ru --endpoint-url=https://storage.yandexcloud.net --profile yc --delete",
    "deploy:only": "aws s3 sync dist/ s3://azoneai.ru --endpoint-url=https://storage.yandexcloud.net --profile yc --delete"
  }
}
```

Теперь:
- `npm run deploy` — собирает проект И загружает в облако
- `npm run deploy:only` — загружает уже собранный `dist/` (если build делался отдельно)

### Шаг 8.2. Ежедневный рабочий процесс

```
1. Редактируешь сайт в Cursor
2. Проверяешь локально: npm run dev
3. Деплоишь: npm run deploy
4. Готово — сайт обновлён на azoneai.ru
```

## Часть 10. Форма обратной связи (замена Netlify Functions)

У Astro-сайта на Object Storage нет серверной логики. Варианты для формы:

### Вариант 1: Formspree (самый простой)

1. Зарегистрируйся на https://formspree.io
2. Создай форму, получи endpoint (вида `https://formspree.io/f/xxxxxxxx`)
3. В Astro-компоненте контактной формы замени action:

```html
<form action="https://formspree.io/f/xxxxxxxx" method="POST">
  <input type="text" name="name" placeholder="Имя" required />
  <input type="email" name="email" placeholder="Email" required />
  <textarea name="message" placeholder="Сообщение" required></textarea>
  <button type="submit">Отправить</button>
</form>
```

Письма будут приходить на email, указанный в Formspree. Бесплатно — до 50 отправок/мес.

### Вариант 2: Yandex Cloud Functions (полный контроль)

Если нужна отправка через SMTP на order@azone-it.ru — можно создать Yandex Cloud Function (аналог Netlify Functions). Это отдельная тема, но архитектурно выглядит так:

```
Форма → fetch('https://functions.yandexcloud.net/xxxx', {POST}) → Cloud Function → SMTP → order@azone-it.ru
```

Могу подготовить код этой функции отдельно, когда дойдёте до этого шага.

### Вариант 3: EmailJS (без бэкенда, из браузера)

1. https://www.emailjs.com — бесплатно до 200 писем/мес.
2. Подключаешь JS-библиотеку, настраиваешь SMTP-шаблон
3. Письмо отправляется прямо из браузера пользователя

## Часть 11. Отключение Netlify

После того как сайт стабильно работает на Yandex Cloud (подожди 2–3 дня):

1. Убедись, что DNS azoneai.ru указывает на Yandex Cloud (не на Netlify)

3. Убедись, что azoneai.netlify.app больше не отвечает (или отдаёт 404)

## Возможные проблемы и решения

**Проблема:** `aws s3 sync` выдаёт ошибку `Access Denied`
**Решение:** Проверь, что у сервисного аккаунта есть роль `storage.editor`, и что ключи в профиле `yc` правильные. Перепроверь: `aws configure --profile yc`

**Проблема:** Сайт по домену не открывается после смены NS-серверов
**Решение:** Подожди 1–2 часа (иногда до 48ч). Проверяй через `dig azoneai.ru` или https://check-host.net/check-dns

**Проблема:** HTTPS не работает, браузер ругается на сертификат
**Решение:** Проверь статус сертификата в Certificate Manager. Он должен быть `Issued`. Если `Validating` — DNS-проверка ещё не прошла. Проверь, что ANAME/CNAME-запись создана правильно.

**Проблема:** Страницы вида `/products/contentguard` выдают 404
**Решение:** Astro по умолчанию генерирует файлы как `products/contentguard/index.html`. Object Storage ищет `index.html` только для корня. Убедись, что в настройках Astro стоит `trailingSlash: 'always'` (или `'never'` — но тогда нужны файлы `.html`). Проверь в `astro.config.mjs`:

```js
export default defineConfig({
  trailingSlash: 'always',  // генерирует /products/contentguard/index.html
  // ...
});
```

Если используешь `trailingSlash: 'never'`, то Astro генерирует `products/contentguard.html` — Object Storage отдаст его по прямому URL, но без `/` в конце. Оба варианта рабочие, главное — быть последовательным.

**Проблема:** Формспри или EmailJS не отправляет письма
**Решение:** Проверь, нет ли ошибок в консоли браузера (F12 → Console). Убедись, что домен добавлен в список разрешённых в настройках сервиса.
